# 🛠️ دليل ربط الموقع بـ Supabase (خطوة بخطوة)

الموقع دلوقتي بقى فيه لوحة تحكم كاملة تقدر منها:
- تضيف/تمسح صور المعرض
- ترفع أو تغيّر الأغنية
- تغيّر باسورد دخول هاجر للموقع

كل ده محتاج مشروع Supabase مجاني. اتبعي الخطوات دي بالظبط:

---

## 1) اعملي حساب ومشروع على Supabase

1. روحي على https://supabase.com وسجلي حساب مجاني (ممكن بجيميل).
2. دوسي **New Project**.
3. اختاري اسم للمشروع (مثلاً `hager-love-site`) وباسورد قوي لقاعدة البيانات (احفظيه في مكان آمن، مش هيتكرر استخدامه هنا لكن مهم).
4. اختاري أقرب Region (مثلاً Frankfurt أو أي منطقة قريبة من مصر).
5. استني دقيقة لحد ما المشروع يخلص إعداد.

---

## 2) شغّلي كود قاعدة البيانات

1. من القائمة الجانبية جوه المشروع، دوسي على **SQL Editor**.
2. دوسي **New query**.
3. افتحي ملف `supabase_schema.sql` الموجود جنب الكود، وانسخي كل محتواه.
4. الصقيه في الـ SQL Editor ودوسي **Run**.
5. لو ظهر "Success" يبقى تمام، الجداول اتعملت.

---

## 3) اعملي Storage Buckets (أماكن تخزين الصور والصوت)

1. من القائمة الجانبية دوسي **Storage**.
2. دوسي **New bucket** واعملي 3 buckets بالأسماء دي *بالظبط*:
   - `gallery-images`
   - `site-music`
   - `voice-messages`
3. عند إنشاء كل bucket، فعّلي خيار **Public bucket** (عشان الصور والصوت يظهروا في الموقع).

> السياسات (Policies) اللي بتحدد مين يقدر يرفع/يمسح اتضافت أوتوماتيك لما شغلتي ملف الـ SQL في الخطوة اللي فاتت.

---

## 4) اعملي حساب أدمن (بياناتك إنت بس)

عشان تدخلي على `/admin` وتديري الموقع، لازم يبقى ليكي حساب دخول:

1. من القائمة الجانبية دوسي **Authentication** > **Users**.
2. دوسي **Add user** > **Create new user**.
3. حطي إيميلك وباسورد قوي (ده اللي هتدخلي بيه على لوحة التحكم).
4. تأكدي إن **Auto Confirm User** مفعّل (عشان متحتاجيش تفعيل من الإيميل).

---

## 5) هاتي مفاتيح المشروع

1. من القائمة الجانبية دوسي **Project Settings** (⚙️) > **API**.
2. هتلاقي:
   - **Project URL** (شكله زي `https://xxxxx.supabase.co`)
   - **anon public key** (مفتاح طويل)
3. افتحي ملف `.env.example` في المشروع، اعملي نسخة منه واسميها `.env`
4. حطي فيه القيمتين دول:

```
VITE_SUPABASE_URL=https://xxxxx.supabase.co
VITE_SUPABASE_ANON_KEY=المفتاح-الطويل-هنا
```

---

## 6) شغّلي الموقع

```bash
npm install
npm run dev
```

الموقع هيفتح على `http://localhost:5173`
لوحة التحكم هتفتح على `http://localhost:5173/admin`

---

## 7) لما تنشري الموقع (Vercel / Netlify)

لازم تضيفي نفس متغيرات الـ `.env` في إعدادات المشروع بتاع الاستضافة:

- **Vercel**: Project Settings > Environment Variables
- **Netlify**: Site configuration > Environment variables

وحطي نفس الاسمين:
```
VITE_SUPABASE_URL
VITE_SUPABASE_ANON_KEY
```

---

## ملاحظات مهمة

- الباسورد الافتراضي لدخول هاجر للموقع هو `123` — غيّريه أول حاجة من لوحة التحكم `/admin` بعد ما تربطي Supabase.
- رابط لوحة التحكم `/admin` مش ظاهر في أي مكان في الموقع، بس لو حد عرف الرابط مش هيقدر يدخل من غير إيميل وباسورد الأدمن.
- لو مربطتيش Supabase لسه، الموقع هيشتغل عادي بالطريقة القديمة (صور من مجلد public/images، وباسورد `123`)، لحد ما تظبطي الإعدادات.
